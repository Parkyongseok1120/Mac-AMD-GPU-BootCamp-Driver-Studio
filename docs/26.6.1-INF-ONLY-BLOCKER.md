# 26.6.1 INF-only blocker

This note records an analysis of the exact official 26.6.1 package verified by
`radeon-pro-5500m-text-only.json`.

## Verified facts

- The unmodified 26.6.1 kernel is
  `B026079/amdkmdag.sys` with SHA-256
  `79AE113DF4EA446C01FA4F3300501D6E6854CB61A4A0F453BF5A7A4767EB1EB7`.
- The unmodified configuration data is
  `B026079/amdgcf.dat` with SHA-256
  `D1AC965FDD33ADE6C7554CA9E3DEF97845E109B7A53B4EE0B8BFCBBC44C68D2A`.
- The INF-only profile preserves both files byte-for-byte. Its only package-content change is the
  INF; it recreates a catalog because the INF change invalidates AMD's original catalog.

## Why the original files fail on the target device

At file offset `0x52A9F` in the official `amdkmdag.sys`, the driver calls an internal system
configuration routine and checks its `NTSTATUS` result. The original conditional branch continues
only when the result is non-negative. The failure path logs this exact string:

`AMDGCF verification failed, driver will fail to start. Forced install, or issue with AMDGCF file?`

The legacy working package changes that conditional branch into an unconditional jump. It also adds
an entry to `amdgcf.dat` and increases its entry count from `0xAA` to `0xAB`. Those are exactly the
two changes an INF-only package intentionally does not make.

Therefore, changing INF hardware matching or setting `PP_Apple_Bootcamp_Enable` cannot make the
unmodified 26.6.1 kernel accept this device when its own AMDGCF verification returns failure.

## Consequence

There is no correct local INF/registry-only repair for this exact 26.6.1 package. A genuine
no-binary-patch 26.6.1 release needs an AMD-issued 26.6.1 package whose original configuration data
and kernel already accept `PCI\\VEN_1002&DEV_7340&SUBSYS_020F106B`, or an AMD-provided supported
override. Microsoft signing can make a modified INF installable, but it cannot change this kernel
runtime rejection.

The INF-only profile remains an experimental diagnostic recipe, not a user-installable release.
