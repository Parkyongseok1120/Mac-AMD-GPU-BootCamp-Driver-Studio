## Confirmed differences

### `amdkmdag.sys`

| Item | 26.6.1 | 25.2.1 |
|---|---|---|
| AMDGCF failure string | **Present** at offset `0x7A27F0` | **Absent** |
| Gate byte at `0x52A9F` | `0x79` (conditional branch before failure path) | Not applicable at same offset; no matching failure string |
| File size | `102,213,136` bytes | `110,955,960` bytes |

The 26.6.1 kernel contains the exact runtime failure string documented in
`docs/26.6.1-INF-ONLY-BLOCKER.md` and checks the internal configuration result at
`0x52A9F`. The legacy working package replaces `0x79` with `0xEB` at that offset.

The 25.2.1 anchor kernel does **not** contain the AMDGCF failure string. This is the
strongest static explanation for why the 25.2.1 WHQL anchor can keep the original
kernel byte-for-byte while 26.6.1 cannot.

### `amdgcf.dat`

| Item | 26.6.1 (`B026079/amdgcf.dat`) | 25.2.1 anchor package |
|---|---|---|
| Present in profile-tracked package root | Yes | No `amdgcf.dat` tracked in `B412641/` |
| Entry count (offset 0 int32) | `170` (`0xAA`) | n/a in tracked anchor files |
| Boot Camp pattern `407340` | **Absent** | n/a |
| Boot Camp pattern `020F106B` | **Absent** | n/a |

The 26.6.1 official `amdgcf.dat` therefore lacks the Boot Camp GPU record that the
legacy binary patch inserts at offset `342` and bumps the count from `0xAA` to `0xAB`.

## Hybrid recipe implication

`original-kernel-hybrid` keeps:

- 25.2.1 original `amdkmdag.sys`
- 26.6.1 original `amdxx64.dll`
- 26.6.1 original `amdgcf.dat`

This is intentional for the first no-binary-modification experiment. The open runtime
question is whether a 25.2.1 kernel accepts the 26.6.1 GCF table without modification.
That question is **not** answered by static analysis alone and requires the hardware E2E
path in `Tools/Test-Whql-Hybrid-26.6.1-25.2.1.ps1`.

## Conclusion

INF/registry/catalog changes alone cannot satisfy the 26.6.1 kernel because:

1. The 26.6.1 kernel contains an AMDGCF verification gate and explicit failure string.
2. The official 26.6.1 `amdgcf.dat` lacks the Boot Camp GPU record targeted by the
   legacy patch.

The 25.2.1 anchor path avoids both blockers by using a different official kernel that
does not expose the same AMDGCF failure gate in static analysis.
