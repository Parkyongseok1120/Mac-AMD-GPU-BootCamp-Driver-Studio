# 26.6.x technical validation notes

Historical record of why 26.6.x no-binary-patch and hybrid recipes did not reach a verified Boot Camp install on the target hardware.

**This document is technical background only.** Distribution of 26.6.x patch workflows was discontinued for **copyright, EULA, and licensing** reasons. See [OFFICIAL-CHANNEL-BACKLOG.md](OFFICIAL-CHANNEL-BACKLOG.md).

Target: MacBook Pro 16-inch 2019, Radeon Pro 5500M — `PCI\VEN_1002&DEV_7340&SUBSYS_020F106B&REV_40`

---

## Static blockers (26.6.1 / 26.6.4)

| Check | Result |
|---|---|
| INF `ExcludeID` for `020F106B` | Present (3 occurrences in 26.6.4 `u0202073.inf`) |
| `amdgcf.dat` Boot Camp record | No `407340` / `020F106B` entry (170 entries in 26.6.4) |
| `amdkmdag.sys` AMDGCF gate | Failure string present; gate byte `0x79` at offset `0x52A9F` |

INF-only modification alone cannot satisfy the 26.6.x kernel runtime AMDGCF check on this hardware.

---

## Runtime results

### 26.6.1 binary-patch (local catalog, TESTSIGNING OFF)

| Step | Result |
|---|---|
| Package prepare | PASS |
| Post-reboot | **FAIL** — CODE=31, `0xC0000428` (`STATUS_INVALID_IMAGE_HASH`) |
| Cause | Locally signed `amdgpu.cat` rejected under TESTSIGNING OFF |

### 26.6.4 kernel-only hybrid (25.2.1 kernel + 26.6.4 UMD/data, TESTSIGNING ON)

| Step | Result |
|---|---|
| Package prepare + local catalog sign | PASS |
| `pnputil /add-driver` | PASS (`3010`, reboot pending) |
| Post-reboot | **FAIL** — CODE=31, `0xC0000182` (`STATUS_DEVICE_CONFIGURATION_ERROR`) |
| Loaded kernel | 25.2.1 original (`E04E8054…`) — signing accepted |
| Rollback | PASS — restored 25.2.1 anchor `32.0.12033.5029`, CODE=0 |

Kernel reached load but failed device initialization when paired with 26.6.4 kernel-side data (`amdgcf.dat`, `amdept.dat`, and other versioned blobs).

### Experiment A — kernel data matching (TESTSIGNING ON)

Goal: keep 25.2.1 kernel + 26.6.4 UMD; align `.dat` files with 25.2.1.

| Experiment | Change | Result |
|---|---|---|
| A-1 minimal | Remove `amdgcf.dat`; replace `amdept.dat` with 25.2.1 | **FAIL** — same `0xC0000182` |
| A-2 full | Replace all shared `.dat` with 25.2.1; remove 26.6.4-only files | **FAIL** — same `0xC0000182` |

Data-file alignment alone did not fix the configuration error.

### Microsoft attestation (not completed)

| Prerequisite | Status |
|---|---|
| Attestation package export | Prepared locally |
| Microsoft-signed `amdgpu.cat` | Not received |
| Expected effect | Catalog trust only — would not fix `0xC0000182` without a CODE=0 runtime recipe |

---

## Technical conclusion

1. **26.6.x INF-only** — blocked by AMDGCF kernel gate and missing Boot Camp GCF data.
2. **26.6.x + local catalog under TESTSIGNING OFF** — blocked by catalog trust (`0xC0000428`).
3. **25.2.1 kernel + 26.6.4 UMD hybrid** — signing OK under TESTSIGNING ON, but runtime `0xC0000182` persisted through full kernel-data matching.
4. **Remaining hypothesis** — UMD–KMD DDI / protocol mismatch between 25.2.1 kernel and 26.6.4 user-mode stack, not solvable by `.dat` substitution alone.

The **25.2.1 WHQL anchor** (INF + catalog only, no binary patches) remains the only verified path tested to CODE=0 on target hardware.

## 한국어 요약

### 정적 차단 요인
- 26.6.x INF에 `020F106B` ExcludeID 존재
- `amdgcf.dat`에 Boot Camp(`020F106B`) 레코드 없음
- 26.6.x 커널 `amdkmdag.sys`에 AMDGCF 런타임 게이트 존재

### 런타임 실패 요약
| 경로 | 결과 |
|---|---|
| 26.6.1 바이너리 패치 + TESTSIGNING OFF | `0xC0000428` (로컬 cat 서명 거부) |
| 26.6.4 하이브리드 (25.2.1 커널 + 26.6.4 UMD) | `0xC0000182` (장치 구성 오류) |
| A-1 최소 데이터 매칭 | 동일 `0xC0000182` |
| A-2 전체 `.dat` 25.2.1 치환 | 동일 `0xC0000182` |
| Microsoft attestation | 미완료; cat 신뢰만 해결, `0xC0000182`는 해결 불가로 판단 |

### 기술적 결론
INF/데이터 파일 정렬만으로는 26.6.x + Boot Camp 조합이 되지 않았고, 25.2.1 커널과 26.6.4 UMD 간 DDI 불일치 가능성이 남음. 실제 CODE=0 검증은 **25.2.1 WHQL 앵커**만 확인됨.
